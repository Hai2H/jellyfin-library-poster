jellyfin-library-poster

项目结构、流程和数据流可视化操作台

总览

这个页面把项目按可操作视图拆开：你可以在左侧切换，从入口流程一路看到模块依赖、文件产物和 Jellyfin/Emby API 边界。

入口 main.py

负责 cron 调度和主业务循环。

核心模块 8

认证、拉库、下载、生成、上传、配置、日志。

默认封面 1920x1080

由 Pillow 合成渐变背景、九宫格海报和标题。

运行模式 cron

首次立即执行，之后按配置定时运行。

项目地图

一眼看懂

1

认证服务器

通过用户名密码换取 Jellyfin/Emby 的用户 ID 和访问令牌。

2

遍历媒体库

拉取媒体库列表，排除配置中不处理的库。

3

下载封面

每个库取最多 9 张主海报，写入 poster/<library>。

4

生成并上传

合成 output/<library>.png，按配置决定是否上传。

主流程

1

启动

python main.py 后导入 config.py，读取 config/config.json。

2

初始化映射

如果 init_template_mapping 为 true，先拉媒体库并补全 template_mapping，然后写回 false。

3

调度

没有 cron 就立即执行一次；有 cron 则首次立即执行，并进入定时循环。

4

处理服务器

遍历 JELLYFIN_CONFIGS，把当前服务器配置写入全局 JELLYFIN_CONFIG。

5

处理媒体库

执行获取媒体库、下载海报、生成封面、可选上传四个阶段。

模块职责

main.py

入口、cron 调度、多服务器和多媒体库循环。

config.py

读取 JSON，维护当前服务器配置和认证状态。

auth.py

调用认证接口，获取 User.Id 和 AccessToken。

get_library.py

获取媒体库列表，并按排除列表过滤。

get_poster.py

获取媒体项、筛选有主图的项目、下载封面。

gen_poster.py

Pillow 图像合成核心，生成最终封面。

update_poster.py

读取输出图并上传回媒体库。

logger.py

彩色控制台日志和文件日志。

生成步骤

1

提取主色

从第一张封面中提取最多 10 个常见颜色，筛选适合做背景的 HSL 颜色。

2

创建背景

生成 1920x1080 左深右浅的渐变背景。

3

排列海报

按 315426987 排序，分成三列，每张缩放为 410x610。

4

视觉处理

圆角、阴影、整列旋转，再粘贴到背景画布。

5

绘制标题

按 template_mapping 和 style_config 绘制中英文标题、阴影和色块。

关键参数

画布1920 x 1080
布局ROWS=3, COLS=3
单张尺寸410 x 610
旋转角度-15.8 度
圆角46.1
输出路径output/<library>.png

配置到行为

配置项影响范围说明
jellyfin[]认证、拉库、下载、上传支持多服务器；每轮处理前把当前服务器写入 JELLYFIN_CONFIG。
cronmain.py控制是否进入定时循环，以及下一次执行时间。
init_template_mappingconfig.py, main.py启动时自动补全媒体库映射，完成后自动写回 false。
exclude_update_libraryget_library.py, main.py过滤媒体库，并在上传阶段再次保护。
style_configgen_poster.py控制中英文字体和文字阴影。
template_mappingget_poster.py, gen_poster.py控制显示名称、英文名、媒体项排序方式。
